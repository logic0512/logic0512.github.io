# 云星 · 自由（Cloud Star · Freedom）

记忆闪光点项目的第 1 颗关键词星球。主题：自由。

每条记录 = 一朵白云。背景是漂动的装饰云，铺满整个天空。拖拽探索云海，点击云朵看念头。

## 入口文件

**`index.html`** —— 用户实际使用的入口
- 启动时从 `localStorage['cloud-star-records']` 读取记录
- 没有记录就显示空状态（中央提示「把你第一个自由的瞬间放进云里」）
- 加 / 删 / 改后自动写入 localStorage

**`index-demo.html`** —— 演示版本
- 内置 12 条 demo 记录，用于截图 / 演示
- 不持久化，刷新即重置
- 通过 `<body data-demo="true">` 标识

## 工具文件

**`tools/picker-decor.html`** —— 装饰云形状选品页
- 30 个候选装饰云形状
- 用户选 10 个 → 这 10 个被嵌入 index.html 的 `DECOR_SHAPES` 数组

**`tools/picker-record.html`** —— 记录云形状选品页
- 30 个候选记录云形状
- 用户选 15 个 → 这 15 个被嵌入 index.html 的 `RECORD_SHAPES` 数组

**`tools/library-preview.html`** —— 当前云形库预览
- 看现在选定的 10 装饰 + 15 记录长什么样
- 用 picker v2 同款渲染参数

## 核心交互

### 添加记录
1. 点右下角 ✦ 按钮
2. 中央浮出输入面板
3. 写文字（≤ 500 字，超过 400 字数变橘色）
4. Cmd/Ctrl + Enter 提交 / 或点「放进云里」
5. 新云朵随机落在当前页的某个空 zone 内

### 查看记录
- 点击任意白云 → 弹卡片显示文字 + 日期
- 卡片右下角「改」按钮可编辑文字

### 删除记录
- 长按白云约 0.6 秒
- 弹出确认框「让这朵云飘走？」
- 点删除 → 云淡出消失

### 探索旧记录
- 第 9 朵起进入第 2 页（往右）
- 第 1 页时右边缘显示「→ 还有 N 朵」提示
- 拖动画面往左 → 看后续页

## 数据结构

每条记录长这样：

```js
{
  text: "辞掉那份工作的下午，走在街上突然很轻松...",  // 用户输入
  date: "2024年5月",                                  // 自动生成
  _id: "r5",                                          // 内部 ID
  _pageIdx: 0,                                        // 所在页（0=主页）
  _zoneIdx: 3                                         // 所在 zone（0-7）
}
```

存到 `localStorage['cloud-star-records']`（JSON 字符串）。

## 已知限制

| 限制 | 数量 |
|------|------|
| localStorage 配额 | ≈ 5MB → 理论 12-25k 条 |
| SVG 渲染流畅 | < 100 条 |
| SVG 渲染可用 | < 300 条 |
| 视觉缩放可读 | < 8 页（即 < 64 条），超过会逐页缩到很小 |
| 装饰云背景覆盖 | 前 2 页有装饰云，第 3 页起背景空荡 |
| 导航 | 只能拖动浏览，无搜索、无年份跳转 |

## 云形库说明

**装饰云**（10 个，铺天空背景）：
- ID 1, 7, 9, 10, 12, 16, 17, 23, 24, 26（从 picker-decor 选）

**记录云**（15 个，装念头载体）：
- ID 1, 4, 6, 9, 12, 13, 14, 16, 18, 23, 24, 26, 27, 28, 29（从 picker-record 选）

每条记录的形状由 `hash(text + date) % 15` 决定，同样的文字 + 日期永远渲染同一形状。

## 渲染参数（重要）

云体 gooey 滤镜：
```
stdDeviation = 6
feColorMatrix alpha = 20 -8
```

阴影 gooey 滤镜：
```
stdDeviation = 5
feColorMatrix alpha = 14 -5
```

BUMP_LIFT = 0.75（鼓包顶离底座中心多远）

**这套参数 picker 和正式版本必须完全一致**，否则同样的 shape 数据会渲染出不同的视觉效果。
