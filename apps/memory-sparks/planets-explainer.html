① 转场动画 · 当前 8 个步骤时序

从「点击星图上的某颗星」开始计时，到「进入星球完全可见」总时长 ≈ 3.2 秒

1. Camera 飞向星球 (2.4s, power2.inOut)
2. flash 径向光出现 (1.1s)
3. iframe fade in (0.7s)
4. flash 淡出 (0.6s)
0s 0.8s 1.6s 2.4s 3.2s

然后 iframe 内介绍卡延迟 2.0s 后从屏幕上方掉下来（在 step3 iframe fade in 期间已经开始），1.0s 用 back.out(1.56) 弹性落定。

你提到的两个问题 + 根因

  • "中间星球抖动太夸张" — 是 step1 的 camera 飞行末段：camera 从 (0,6,16) 加速冲向目标星球 (planet.x, 0, planet.z)，最后阶段进 sprite 内部时 sprite 在视野里**急剧放大**，给人"星球被推到脸上"的冲击感。
    曲线是 power2.inOut（先加速后减速），但减速段在 70%-100% 之间，那段视觉上仍很猛。
  • "星球半透明叠在彩色页" — step2-3 的并行：flash 径向光在 1.6s 开始，iframe fade in 在 2.2s 开始（在 flash 还没淡完时叠加）。这中间 0.6s 里 iframe 透明度从 0 渐到 1，**但背后的 3D 星球场景仍在渲染**，所以你看到的是「半透明 iframe + 透明星球 sprite + 渐弱 flash」三层叠加。
    根本原因：R3F Canvas 在 entering 阶段没有「淡出 sprite」这一步。

可选的优化方向（你来选）

A. 弱化抖动 — camera 飞行参数调整

  • 把 camera 飞行从 2.4s → 3.0s，曲线换成 power2.out（单边减速，末段更柔）
  • 飞行终点不再贴 sprite 表面（surfacePos planet.size * 0.6），改成 planet.size * 1.5，离星球远一点
  • 结果：进入感更"飘进去"而不是"撞进去"

B. 干净的过渡 — 星球完全消失后再显示 iframe

  • step2 flash 出来后，**同时**把所有 sprite + 其他星淡出 (opacity 1→0)
  • flash 全屏白光（不再是径向）覆盖 200ms 完全遮挡
  • iframe 在 flash 全白时 fade in（用户看不到 sprite 叠加层）
  • 结果：白光闪过 → 进入星球界面，无视觉破绽

C. 用 GSAP 平滑接力

  • camera 飞行末段不再加速冲撞，最后 0.5s 用 GSAP 让星球 sprite 加速放大占满全屏 → fade out
  • iframe 在 sprite 占满时立即可见（无需 flash）
  • 结果：星球本身就是"过渡"，没有中间的闪光层

我建议 A + B 结合做。要做就告诉我，5-10 分钟改完。
